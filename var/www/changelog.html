Version 3.3.4 bullseye

System

  • Pi OS upgrade from buster to bullseye.
  • Pi-Ager Auto Hotspot Mode implementiert. Pi-Ager funktioniert damit auch ohne Router und ohne Internet Verbindung. Hat der Pi-Ager keine Verbindung zum WLAN Router oder Access Point, schaltet der Pi-Ager in den Hotspot Mode. Er kann dann via Smartphone oder Tablet erreicht werden. Der Hotspot hat die WLAN SSDI 'RPiHotspot', das Passwort ist '1234567890'. Den Pi-Ager kann man dann über die IP Adresse 10.0.0.5 via SSH ereichen, den Webserver erreicht man mit einer beliebigen IP Adresse, aber nicht aus dem Adressraum http://10.0.0.x. Die Überprüfung, ob eine Verbindung zum Internet oder Access Point besteht und eine Umschaltung der WLAN Betriebsart erforderlich ist, erfolgt alle 2 Minuten.
  • Im Hotspot Mode können auf der ADMIN Seite Datum, Uhrzeit, WLAN SSID und WLAN Passwort gesetzt werden.
  • Reifetabellen: die Spalte 'Tage' wurde durch 'Stunden' ersetzt. Die Eingabe ist limitiert auf Werte zwischen 1 ... 240h. Vor dem Upgrade auf 3.3.4 wird empfohlen, die eigenen Reifetabellen zu exportieren und mit excel oder einem Text-Editor die Spalten-Überschrift 'days' in 'hours' umzubenennen und die Werte in der Spalte 'hours' dann auf Stunden umzurechnen. Diese so modifizierten Reifetabellen können mit der Version 3.3.4 wieder importiert werden.
  • Kühler- und Heizungs-Regelung arbeitet mit getrennten und symmetrischen Hysteresen. Wenn ein externer Sensor angeschlossen ist, wird die Temperatur des externen Sensors in die Regelung einbezogen.
  • Automatisches Abtauen erst möglich, wenn die Soll-Temperatur kleiner als 8° definiert ist.
  • Automatisches Abtauen wahlweise mit oder ohne Umluft aktiv.
  • Erster Pi-Ager Bootup nach dem Aufspielen eines neuen Image beschleunigt. Nicht benutzten man-db Service entfernt. Build Nummer in DB integriert.

Version 3.3.3

System

  • In diese Version sind zahlreiche Optimierungen und Erweiterungen eingearbeitet worden:
  • Ein Bluetooth Temperatursensor 'Mi Temperature and Humidity Monitor2' als externer Sensor wird unterstützt. Dieser Sensor kann auch mit '1-Wire' internen Sensoren kombiniert werden.
  • Die Seiten index.php und diagrams.php wurden vollständig mit AJAX und JQUERY überarbeit, so dass kein manueller oder automatischer Refresh der ganzen Seiten mehr erforderlich ist. Es werden lediglich die Messwerte, Charts und das Statusbord aufgefrischt.
  • Diese Maßnahmen reduzieren die Systemlast und verbessern die Bedienbarkeit erheblich.
  • Die Einstellungen, welche Kurven (Feuchte, Temperatur) in den Charts sichtbar sind, bleiben auch nach Seitenwechsel erhalten. Ausnahme: Safari in mobile IOS
  • Pi-Ager started nach shutdown/Neustart mit den zuletzt eingestellten Werten. Bei einer Exception startet das System mit dem Status Pi-Ager gestoppt.
  • Änderungen der Umluft- und Abluft-Parameter werden sofort übernommen und nicht erst nach Ablauf des aktuellen Zyklus.
  • Abluft Ventilator wird nur aktiviert, wenn die externe absolute Feuchte niedriger ist als die interne absolute Feuchte. Funktioniert nur, wenn ein externer Sensor angeschlossen ist.
  • Der 'Schalter' Eingang kann nun benutzt werden, um die Ausgänge 'Licht' und 'UV-Licht' mit Vorrang vor den Timern und anderen Einflüssen direkt anzusteuern. Zum Beispiel kann damit das UV-Licht ausgeschaltet werden, wenn der Türschließkontakt des Reifeschranks mit dem 'Schalter'-Eingang auf der Pi-Ager Platine verbunden wird.
  • Automatisches Abtauen des Reifeschranks ist nun implementiert. Der Vorgang kann mit folgenden Parametern gesteuert werden: Abtauen aktiv, Abtauzyklus und Abtautemperatur.
  • Neue Events eingefügt: Abtauen gestartet und Abtauen beendet.
  • Im Log werden nur noch Änderungen der Ausgänge aufgezeichnet und rot markiert.
  • Diverse Bugs gefixed:
  • Kurzzeitige Aktivierung der Relais beim Neustart
  • Diagramm für Waage 2 nach kurzer Zeit verschwunden
  • Bug fixed: Benutzerdefinierter Timeframe
  • Issue fixed: Verzögerungszeit beim Wiedereinschalten des Kompressors
  • Bug fixed: Berechnung der Temperatur und Feuchte bei den Sensoren SHT3X und SHT85: die Werte wurden jeweils um 1°C bzw. 1% (Feuchte) zu hoch angezeigt.
  • Bug fixed: Die Buttons 'Zurück' und 'Weiter' auf der Seite logs.php funktionierten nicht, wenn nur ein Log File (logfile.txt) vorhanden war.
  • Die Timer Parameter 'Dauer' und 'Periode' wurden umbenannt: 'Dauer' -> 'EIN Dauer', 'Periode' -> 'AUS Dauer'.
  • Die Waagenwerte größer 9999gr auf dem TFT Display wurden nicht vollständig angezeigt. Das Problem kann durch eine verbesserte Firmware behoben werden. Hierzu einfach die Nextion Display Firmware neu flashen.
  • TFT Display timeout voreingesteller Wert ist jetzt 60s.
  • Umluft Steuerung: ein Schalter in den allgemeinen Einstellungen ermöglicht die Kopplung der Umluft mit dem UV-Licht.
  • Zur Überwachung eines möglichen Fehlverhaltens des Kompressor-Relais (Schaltkontakt schaltet nicht ein/aus) wurde die Software ereitert. Benötigt wird ein Stromsensor LEM HO 6-P/SP33, der an den 4. Sensor-Eingang der Pi-Ager Platine angeschlossen wird. Siehe auch die Hilfe zu den Thermometern auf der ADMIN Seite.

  • Bug fix für Nextion TFT HMI Displays: wenn das Kühlschrank-Symbol bzw. das Licht Icon aktiviert wird, wird das Licht für 10 Minuten eingeschaltet.

  • Auf dieser Seite kann auch der Pi-Ager gestartet und gestoppt werden.

  • Wichtig:
  • Die im Pi-Ager Image hinterlegte aktuelle Display Firmware muss nochmals geflasht werden, damit die oben aufgeführten Features funktionieren. Hierzu muss auf der ADMIN Seite in der Sektion 'NEXTION DISPLAY FIRMWARE' erst das angeschlossene Display Device ausgewählt und gespeichert werden. Anschließend muss der Button 'Programmierung starten' aktiviert werden.

Administration

  • NFS Backup wurde überarbeitet. Die einzustellenden Parameter wurden auf das unbedingt nötige Maß reduziert. Das Backup Log kann nun in einem Browser Tab angezeigt werden. Probleme mit dem Backup können dort analysiert werden.

Nachrichten

Events

  • Überwachung der internen Temperaturmesswerte, Min- und Max-Werte können eingestellt werden und können bei Unter-bzw.-Überschreitung ein Event auslösen.
  • Überwachung der UPS Batterie-Spannung und der Versorgungsspannung. UPS Batterie low und Powerfail können Events auslösen.
  • Überwachung des Schalter-Eingangs. Schalter offen/geschlossen kann ein Event auslösen.
  • Achtung: die Überwachung der o.g. Messwerte erfolgt in einem 10s Zeitraster.
